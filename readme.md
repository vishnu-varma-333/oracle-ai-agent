# 💎 Oracle Intelligence Pro: Agentic SQL Analyst
### *Autonomous Natural Language Interface for Enterprise Databases*

## 🚀 Overview
**Oracle Intelligence Pro** is a sophisticated **Agentic AI system** that allows users to query an Oracle Database using plain English. 

Unlike traditional "Text-to-SQL" scripts, this project utilizes a **Self-Healing Agentic Workflow** built on **LangGraph**. When the AI generates a SQL query that results in a database error (e.g., syntax issues or schema mismatches), the agent captures the Oracle `ORA-XXXXX` error, reasons through the failure, and **automatically regenerates the query** until the correct data is retrieved.

---

## 🧠 Agentic Architecture
This system is powered by a directed acyclic graph (DAG) that manages the state of the conversation and the database interaction:

1.  **SQL Planner (The Architect):** Interprets the natural language question and maps it to the database schema to generate an Oracle-specific SELECT query.
2.  **DB Executor (The Operator):** Interfaces with the Oracle 23ai Docker container using the `oracledb` driver.
3.  **Self-Correction Loop (The Guardrail):** If the database returns an error, the agent increments a revision counter and loops back to the Planner with the error logs for an immediate fix.
4.  **Response Synthesizer (The Analyst):** Transforms raw data rows into a polished, human-friendly business insight.

## 🛠 Tech Stack

## ⚙️ Setup & Installation

### 1. Prerequisites
*   **Docker Desktop** (To run the Oracle instance)
*   **Python 3.10+**
*   **Google Gemini API Key**

### 2. Database Initialization
Run the following to start your local Oracle instance:
```bash
docker run -d -p 1521:1521 --name oracle-db \
-e ORACLE_PASSWORD=YourPassword123 \
gvenzl/oracle-free
```

Once the container is ready, execute these SQL commands to set up the demo environment:
```sql
CREATE TABLE STUDENTS (ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, NAME VARCHAR2(100), AGE NUMBER, DEPARTMENT VARCHAR2(50));
CREATE TABLE MARKS (MARK_ID NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, STUDENT_ID NUMBER REFERENCES STUDENTS(ID), SUBJECT VARCHAR2(50), SCORE NUMBER);

INSERT INTO STUDENTS (NAME, AGE, DEPARTMENT) VALUES ('Vishnu', 21, 'CS');
INSERT INTO STUDENTS (NAME, AGE, DEPARTMENT) VALUES ('Chanakya', 22, 'EC');
INSERT INTO STUDENTS (NAME, AGE, DEPARTMENT) VALUES ('John', 20, 'CS');

INSERT INTO MARKS (STUDENT_ID, SUBJECT, SCORE) VALUES (1, 'Maths', 95);
INSERT INTO MARKS (STUDENT_ID, SUBJECT, SCORE) VALUES (2, 'Maths', 28);
INSERT INTO MARKS (STUDENT_ID, SUBJECT, SCORE) VALUES (3, 'Maths', 30);
COMMIT;
```

### 3. Application Setup
```bash
# Clone the repository
git clone https://github.com/vishnu-varma-333/oracle-ai-agent.git
cd oracle-ai-agent

# Install dependencies
pip install langgraph langchain-google-genai oracledb streamlit python-dotenv
```

### 4. Environment Configuration
Create a `.env` file:
```env
GOOGLE_API_KEY=your_gemini_api_key
DB_USER=system
DB_PASSWORD=YourPassword123
DB_DSN=localhost:1521/FREE
```

## 🛡 Features & Safety
*   **Autonomous Error Correction:** Automatically handles ORA-errors without user intervention.
*   **SQL Scrubbing:** Prevents common AI formatting errors (markdown backticks, semicolons) from reaching the database.
*   **Schema Discovery:** Dynamically reads the Oracle Data Dictionary (`user_tab_columns`) so it can work with any table you add later.
*   **Premium UX:** Includes a real-time workflow visualizer and a technical drill-down for SQL transparency.
